While this guide describes the process of contributing code,
documentation, and tests to PlasmaPy, there are many other ways to
contribute. Some of the many possibilities are to:

* Contribute new code, documentation, and tests.
* Refactor existing code and tests.
* Improve the description of plasma physics in `PlasmaPy's documentation`_.
* Write educational Jupyter_ notebook that introduce plasma concepts
  using PlasmaPy.
* Create videos that show how to use PlasmaPy. üé•
* Participate in code reviews.
* Help with project management.
* `Request new features`_.
* `Report bugs`_. üêû
* Improve `PlasmaPy's website <PlasmaPy>`_.
* Help organize events such as `Plasma Hack Week`_. üìÜ
* Provide feedback on how existing functionality could be improved.
* Help update PlasmaPy's development roadmap. üõ£Ô∏è
* Be part of the PlasmaPy community!

during PlasmaPy's weekly `office hours`_.
